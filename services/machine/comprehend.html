Resumo Arquitetural – Amazon Comprehend

Amazon Comprehend é um serviço de Processamento de Linguagem Natural (NLP) totalmente gerenciado que permite extrair significado de textos não estruturados usando aprendizado de máquina, sem necessidade de treinamento prévio.

Principais Funcionalidades

  • Detecção de idioma
  • Análise de sentimento (positivo, negativo, neutro, misto)
  • Reconhecimento de entidades (nomes, datas, lugares)
  • Extração de frases-chave
  • Análise de tópicos

Segurança

  • Acesso controlado por IAM roles e policies
  • Dados em trânsito: TLS 1.2
  • Dados em repouso: criptografia com KMS (inclusive BYOK)
  • Suporte a VPC Endpoints (PrivateLink)

Integrações Comuns

  • S3: Entrada e saída de arquivos (batch)
  • Lambda / Step Functions: Processamento automatizado e orquestração
  • EventBridge: Acionadores por evento
  • SageMaker: Análise e pós-processamento
  • Kendra: Enriquecimento de buscas

Custos

  • Real-time (síncrono): ~$0.0001 por unidade (100 caracteres)
  • Batch: $1/job + $1/100K caracteres

Escalabilidade

Totalmente serverless, com escalabilidade horizontal automática.

Mini-cenário de integração

Chatbot jurídico detecta sentimentos e entidades antes de passar conversa ao atendente humano:

Lex → Lambda → Comprehend → EventBridge → SNS/Slack

Dicas de Arquiteto Sênior

  • Use VPC Endpoint + KMS sempre que houver dados sensíveis
  • Para interações curtas, use modo síncrono; para grandes volumes, use batch
  • Prefira Step Functions para pipelines mais robustas com fallback
  • Agrupe textos pequenos para otimizar chamadas e reduzir custos
